Cordova + Framework7 + Vue 简单总结

发表于 2018-01-09

摘要

前段时间在做CFV项目过程中，遇到了一些问题，现简单收集整理如下：

Part 12

1、点击空白部分界面消失

通常做法是做一个遮罩层，在遮罩层上添加方法，F7提供了各种弹窗操作，可以借助完成，还可以借助Cordova，调用原生处理。

2、v-if和v-show的区别

需要频繁切换用v-show。

  • v-if 的初始化较快，但切换代价高; v-show初始化慢，但切换成本低。
  • v-if是动态的向DOM树内添加或者删除DOM元素，有更高的切换消耗。
  • v-show是通过设置DOM元素的display样式属性控制显隐，更高的初始渲染消耗。

3、fixed和absolute定位区别

fixed:固定定位,absolute:绝对定位

区别：

  • 没有滚动条的情况下没有差异
  • 在有滚动条的情况下，fixed定位不会随滚动条移动而移动，而absolute则会随滚动条移动

做键盘底部弹出，让输入框跟随移动，用fixed把输入框定在底部，但是ios手机下用户用第三方键盘，输入框会被遮挡，期间采用了cordova的键盘处理插件，但是效果不好，最后采用的方案是弹出新页面操作。

4、Vue 的数据通信

  • 顺传，用property即可。
  • 逆传，提前给A页面绑定执行的方法，B页面emit执行方法即可。
  • 跨层，提前注入需要执行的方法，用emit。

5、关闭eslint检测

webpack.base.conf.js 配置文件中的eslint rules注释掉即可。

6、修改Framework7 默认文字显示

在初始化Framework7 的时候，查看F7的文档，取得属性值，修改即可。

7、Vue 设置全局变量

global.属性

8、VSCode 断点调试

在webpack.base.conf.js 的 publicPath 新增

devtoolModuleFilenameTemplate: ‘../[resource-path]’

配置launch.json

9、真机调试问题

cordova + f7 调试流程比较麻烦，每次修改都要build, 可以用 http-server mock 模拟数据结构。

10、查看端口占用情况

lsof -i :端口号

kill 端口号

11、Vue加载失败显示默认图片

img标签绑定 :onerror = ‘图片’
